Browser LLM Runner Help

Browser LLM Runner is a local, student-facing chat app that runs supported language models in your browser. It is designed as both a chat tool and a learning tool: model behavior is visible, settings use plain-language labels, and conversations can be copied, revised, compared, and exported for study. This guide covers the full workflow: starting a conversation, loading a model, streaming and stopping responses, editing branches, exporting conversations, and recovering from common issues.

Start and load a model

  1. Open the app and select Start a conversation from the home screen.
  2. In the pre-chat panel, choose a supported model from Choose a model.
  3. Write your first prompt in Message input and send it.
  4. The app loads the selected model only when you send the first message. Progress appears in the status area while the engine initializes.
  5. After loading finishes, the full chat controls appear and the response begins streaming into the transcript.
If you already selected a saved conversation before loading a model, the pre-chat panel will prompt you to load the model first. This is expected behavior.

Chat, stop, and review responses

During generation

  • Responses stream into the transcript incrementally for faster feedback.
  • Use Stop generating at any time to cancel the current response.
  • If you change token or sampling settings during generation, the app queues those changes and applies them after the current response finishes or stops.

After generation

  • Model replies support Copy, Regenerate, and Fix.
  • When a model outputs internal thinking tags, the app places that content in a separate Thinking section instead of mixing it into the final answer.
  • Markdown formatting and LaTeX math render directly in the transcript.
What does Fix do?

Fix runs a multi-step orchestration that critiques the current response, revises it, and validates the revised answer before adding the result as a new response variant at the same turn.

Conversations, branches, and variants

Conversation management

  • Conversations are saved locally and restored when you reopen the app.
  • New conversations start untitled and are automatically renamed after the first completed reply.
  • After that first completed reply, a download menu appears so you can export the visible branch as Markdown or JSON.
  • Each conversation can store its own system prompt. You can append it after the captured default prompt or use it instead.

Branch-aware editing

  • Edit updates a user message in place and removes later turns on that branch.
  • Branch lets you save an alternate user-message path without replacing the original branch.
  • Regenerate creates additional model response variants for the same turn while keeping prior variants available.
  • Left and right branch controls, along with the x/y indicator, let you switch among sibling user branches and model variants.

Settings reference

Backend and model behavior

  • Auto tries WebGPU first, then WASM, then CPU.
  • WebGPU only requires browser WebGPU support.
  • WASM only forces a software fallback path.
  • The selected backend and model are stored in browser storage.

Generation controls

  • Maximum output tokens sets the response length cap.
  • Context size controls short-term memory for the current conversation.
  • Temperature shifts creativity versus predictability.
  • Top K narrows sampling to the most likely next-token options.
  • Top P widens or narrows nucleus sampling for more or less variety.

Per-model controls

  • Token limits and temperature are constrained by the selected model.
  • Those values are disabled until a model is loaded.
  • Context size and temperature each include Reset to model default.
  • Token and temperature overrides are remembered separately for each model.

Conversation controls

  • Default system prompt applies only to newly created conversations.
  • Show thinking controls whether thinking sections start expanded.
  • Enable single-key transcript shortcuts toggles focused transcript keys like E, B, R, F, and C.
  • Transcript view lets you switch between standard and compact layouts.

Using it as a learning tool

Why this app is useful for learning

  • The app exposes more of the model workflow than most chat tools, so you can see how loading, streaming, stopping, revision, and branching actually work.
  • Settings use plain-language labels such as Temperature (Creativity), Top K (Predictability), and Top P (Strangeness) so learners can connect technical controls to practical effects.
  • Response variants and branch-aware editing make it easy to compare prompt phrasing, inspect how outputs change, and discuss why one version is stronger than another.
  • Thinking sections, when available for a model, are separated from the final answer so students can inspect process signals without losing the polished response.

Pedagogical workflows

  • Use Copy on individual prompts, answers, or thinking blocks to move examples into notes, assignments, or teacher feedback documents.
  • Use Regenerate and Fix to compare drafts, revisions, and self-correction behavior on the same task.
  • Export the visible branch as Markdown or JSON when you want a record of a tutoring session, writing exercise, or prompt experiment.
  • Keep prompts local by default, which makes classroom experimentation easier when privacy and careful handling of student writing matter.
A strong classroom pattern is to vary one setting or one prompt at a time, then use branching, regeneration, copying, and export to compare results side by side.

Accessibility and focus behavior

  • Primary controls are reachable with Tab and Shift+Tab.
  • Dialogs and panels close with Escape and return focus to a sensible trigger.
  • The transcript has helper links near the start and end so you can jump to the transcript boundaries or back to the message composer.
  • Status updates such as loading, ready, generating, stopped, and errors are announced through dedicated status regions instead of streaming every token to screen readers.
  • Messages use clear speaker labeling so the conversation stays readable in assistive technologies.

Privacy, storage, and exports

Stored in your browser

  • Selected model and backend preference.
  • Saved conversations and their branch structure.
  • Model downloads cached for reuse.
  • Generation settings such as token limits, temperature, Top K, and Top P.

Export options

  • JSON (.llm.json) exports structured conversation metadata and exchanges.
  • Markdown (.md) exports a readable branch transcript.
  • Exports include only the currently visible branch, not every hidden branch or variant.

Troubleshooting

The model does not load or WebGPU fails
  • Open Settings and switch the backend to WASM only.
  • Send another message to trigger a fresh model load with the new backend.
  • Check the debug info section in Settings for backend and initialization details.
Generation feels stuck
  • Use Stop generating to cancel the current run.
  • Try a shorter prompt or lower the context size before sending again.
  • If you changed settings during generation, wait for the current run to finish or stop first.
I cannot edit token or temperature fields
  • Those controls stay disabled until a model is fully loaded.
  • After loading, the allowed range depends on the selected model.
Keyboard shortcuts are not working inside the transcript
  • Make sure a transcript message has focus.
  • Confirm Enable single-key transcript shortcuts is still turned on in Settings -> Conversation.
